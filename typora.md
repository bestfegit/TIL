# Unix/Linux 명령어

* 현재위치의 폴더, 파일 목록보기 : ls
* 길 위치로 이동하기 cd <path> / <path>는 '절대경로'와 '상대경로'로 나뉨, 절대경로는 변하지 않음 - c드라이브 부터 내가 있는 폴더까지를 말함 / 아래에 있는 폴더로 이동하고 만약 그 폴더가 없다면 오류가 뜨게 됨
*  . : 한개는 내가 있는 위치를 말함
* cd .. : 상위 폴더로 이동 - 상대경로 - 내가 지금 있는 파일에 따라 상위로 가는 결과가 달라짐
* cd ../.. : 두번 위로감
* cd. : 현재 위치로 이동
* 폴더 생성 :  mkdir <name> 
* 파일 생성 :  touch <name>
* 삭제 rm <name>
* 폴더 삭제 rm -r <name>
* 깨끗한 화면 clear

# Git 기본기

### Repsitory 

* git init 명령어로 로컬 저장소 생성

* .git 디렉토리에 버전관리에 필요한 모든 것이 들어있다.

* git status 현재 상태확인

* git add 뒤에 나오는 파일을 스테이징 에어리어로 올림

* git add . '.'은 현재 위치를 말함 <- 한칸 띄우고 . 찍기 - 현재 폴더 내 모든 파일을 스테이징 에어리어에 올림

* git commit -m "commit_message" 스테이지 올린거만 됨 : -m 은 메시지를 보내겠다는 뜻, 최대한 자세하게 적어줘야 무슨 작업을 한 파일인지 알수있음 ex 로그인 구현, 패스워드 구현

* git log 모든 커밋 출력

* git diff 두개의 커밋간의 차이를 보여줌

* git remote add origin {}  주소

* git push -u origin master 관례적으로 마스터 브랜치에 푸시해줌 / -u는  set up stream 이 명령어 이후로는 뒤를 생략하고  git push만 입력해도 작동함 / 클론으로 가져왔으면 로컬 저장소가 있으므로 안해줘도됨

* git clone {} : 리모트의 것을 local로 복사 레파지토리 저장소 가져옴

* git pull origin master : 리모트의 최신 커밋을 로컬로 땡겨옴 / 관례적으로 master 브랜치에서 땡겨오는거

  -evertying up to date = 이미 최신화다 더 못 푸시한다
  
* git restore --staged 파일 : 스테이지에 add하는 걸 취소하기

* git restore 파일명 : 워킹디렉토리에 있는걸 모두 취소하기

* git reset -- hard  파일 : 되돌리기! 

* 폴더명 .gitignore 하고 파일명 등록하면, 원하지않는 파일 제외함. 흐릿하게 바뀜

  -빅데이터 머신러닝은 자료가 많아 소스코드만 깃헙에 올리는 경우가 많음

  - 폴더명/ : 폴더 아래 모든 것을 포함하기

  

* 워킹 트리 클린 : 편집이 이루어지지않아 이전과 같은 상태이다
* 터미널 commit 오른쪽 숫자 4개는 깃헙에 올라간 커밋의 고유 ID

* 협업하는 법  repo 생성 -> 설정 -> 매니지 -> 상대방 초대 -> 이메일 수락

1 Working Directory - git으로부터 감시받지않는 상태

2 Staging Area - git added 하여 이 파일 버전을 무대 위로 올리기 / 관리할 것만 올리는 것

3 Repository - 깃 commit을 해주고 Repository에 넣어줌



리셋 명령문 뒤  - 잘안씀

|       | wd   | sa   | re   |
| ----- | ---- | ---- | ---- |
| soft  | x    | x    | o    |
| mixed | x    | o    | o    |
| hard  | o    | o    | o    |

* o : 과거의 커밋 모습으로 돌아갑니다
* x : 현재작업모습을 남겨둠



브랜치 - 특정 커밋을 가리키는 포인터

 git branch 현재 브랜치 이름을 알려줌

git branch 이름 : 새 브랜치 생성

git checkout 브랜치 이름 : 그 브랜치로 바꿈

git branch -d 이름 : 브랜치 삭제



* Merge : 세컨드 브랜치 작업후 다시 메인 브랜치에 합치는 거

git merge 브랜치 이름 : 그 브랜치를 주 브랜치에 합병

애드랑 커밋을 다시 해줄 필요는 없으나 푸시는 다시 해주어야 업데이트가 됨
